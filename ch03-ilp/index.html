第3章 指令级并行及其利用 (ILP)

指令级并行（Instruction-Level Parallelism）是高性能处理器的核心。从 5 级流水线到超标量乱序执行，ILP 的挖掘是架构演进的主线。

ILP 的概念与挑战

三种数据依赖：RAW（真依赖，必须满足）、WAR/WAW（名称依赖，通过寄存器重命名消除）。
基本块 ILP 有限（平均 4-7 条指令/基本块），需跨基本块调度 + 分支预测。

高级分支预测

预测器准确率说明
2-bit 饱和计数器80-85%4 状态 FSM
相关预测器 (m,n)~92%用 m-bit 历史选 n-bit 预测器
竞赛预测器~95%本地+全局，选择器决定
TAGE>95%多表不同长度历史 + tag

BTB（Branch Target Buffer）：在 IF 级即可获取分支目标 PC，无需等到 EX 级。

Tomasulo 算法

核心：寄存器重命名 + 数据流驱动 + 公共数据总线（CDB）广播。

结构功能
保留站等待发射指令队列
寄存器别名表（RAT）逻辑→物理寄存器映射
CDB结果广播总线
ROB重排序缓冲，顺序提交

执行四阶段：Issue → Execute → Write Result → Commit（ROB 保证精确异常）

SMT (Simultaneous Multithreading)

单周期多个线程的指令同时发射。Intel Hyper-Threading = 2-way SMT。性能提升 15-30%。

ILP 墙

2000 年代后单核 ILP 收益递减：发射宽度 >4 后边际收益急剧下降。性能增长从 52%/年降到 3.5%/年。

你在综合时体会到的「4 发射核时序难收」就是 ILP 墙的物理体现。现代处理器靠多核 + DSA 而不是更宽的发射来提升性能。
