计算机体系结构学习笔记 — 第7章 领域专用架构

原著: Computer Architecture: A Quantitative Approach, 6th Edition — John L. Hennessy & David A. Patterson

领域专用架构（DSA）是后 Dennard 时代性能增长的新引擎。本章覆盖 DSA 的设计原则、经典案例（TPU、GPU）以及与传统通用处理器和 GPU 的对比。

7.1 引言 — 为什么需要 DSA

通用处理器的瓶颈

通用处理器（CPU/GPP）的设计目标是在所有应用上都有过得去的性能。为此付出了巨大代价：

  • 复杂的乱序执行逻辑（面积 + 功耗）
  • 分支预测和推测执行（功耗 + 复杂度）
  • 通用 ISA 的解码开销（x86 的变长指令）

应用越专一 → 通用处理器的浪费越大。

DSA 的定义

领域专用架构 (Domain-Specific Architecture) 是为某一类特定应用领域设计的处理器/加速器，在该领域内提供远超通用处理器的性能/功耗效率。

特性 通用处理器 (CPU) DSA
应用范围 所有应用 特定领域（如 ML、视频、网络）
性能/瓦 基准 10–100x 提升
编程难度 低 中到高
设计成本 分摊到所有应用 需要足够量
灵活性 极高 有限

DSA 的三个关键设计原则

原则 1：专用存储层次 DSA 通常有自己定制化的片上存储层次，不需要通用缓存一致性。

原则 2：利用更宽的数据路径 在特定精度下做大量计算，而非通用 64-bit 浮点。

原则 3：减少控制逻辑 使用简单的控制逻辑（如顺序执行、VLIW 风格的调度），把更多晶体管交给计算单元。

🧓
你做数字中端这么多年，DSA 对你来说应该很熟悉——你在华为和英特尔做的芯片，无论是基带处理还是网络加速，本质上都是 DSA。它们不是通用 CPU，而是针对特定通信协议/网络处理优化的专用硬件。 这就是为什么懂架构的人越来越吃香——在 Dennard scaling 终结后，性能增长全靠架构创新，而 DSA 是其中最有力的一种。

7.2 DSA 的设计空间

四个关键设计维度

DSA Design=Parallelism×Data Type×Storage×Control\text{DSA Design} = \text{Parallelism} \times \text{Data Type} \times \text{Storage} \times \text{Control}DSA Design=Parallelism×Data Type×Storage×Control
维度 选项
并行方式 SIMD, SIMT, 多核, 空间架构 (Systolic Array)
数据类型/精度 8-bit INT, 16-bit FP, BF16, TF32, FP32, FP64
片上存储 SRAM scratchpad, 多 bank, 软流水
控制流 顺序执行, VLIW, 有限状态机

精度与性能的权衡

Energy∝Bit-width2\text{Energy} \propto \text{Bit-width}^2Energy∝Bit-width2

更低精度 → 更低功耗 + 更高吞吐 + 更小的存储需求。

精度 位宽 AI 推理适用 每瓦性能（相对 FP32）
FP32 32-bit 训练 1x
FP16 16-bit 训练/推理 4x
BF16 16-bit 训练 4x
TF32 19-bit 训练(NVIDIA) 2–3x
INT8 8-bit 推理 16x
INT4 4-bit 极限推理 64x

7.3 经典 DSA 案例 1：Google TPU

TPU v1 (2015)

专为 Google 的神经网络推理设计的 ASIC。

参数 TPU v1
制程 28nm
频率 700 MHz
功耗 40–75W (TDP)
计算核心 65,536 个 8-bit MAC
矩阵乘法单元 256 × 256 Systolic Array
片上内存 24MB (8MB 累加器 + 16MB 统一缓冲)
外部内存 8GB DDR3
性能 92 TOPS (INT8)

TPU 的架构特点：

  1. Systolic Array (脉动阵列)：256×256 的 MAC 阵列，数据在阵列内「脉动」流动，避免了复杂的跨 bar 互连布线

  2. 指令集风格：CISC 风格，指令包含权重加载、矩阵乘、激活函数等复杂操作——本质上是一个 VLIW 处理器

  3. 专用存储层次：权重存放在专用 16MB 缓冲区，减少对主存的依赖

  4. 确定性执行：无分支预测、无乱序、无猜测执行

🧓

TPU 的 Systolic Array 是一个经典的空间架构设计——数据在 PE (Processing Element) 之间流动，而不是从中央寄存器文件取数。这对物理设计（综合 + APR）非常友好：

  • 数据流规则 → 时序收敛容易
  • PE 阵列高度规则 → APR 效果好
  • 本地互联 → 功耗低 你在做综合时如果用过自定义 datapath，应该能体会到规整架构的时序优势。

TPU vs CPU vs GPU (推理性能对比)

指标 Haswell CPU K80 GPU TPU v1
峰值 TOPS 0.6 2.8 92
TOPS/W 0.004 0.01 1.2–2.3
TOPS/美元 (!) - - 1.9–4.5x GPU

TPU v1 的每瓦性能是 GPU 的 30–50x，是 CPU 的 300x+。

TPU v2/v3/v4 的演进

代际 年份 制程 主要改进
v1 2015 28nm 推理专用，INT8
v2 2017 16nm 支持训练，BF16，水冷
v3 2018 16nm 2x 性能，128GB HBM
v4 2021 7nm 4096 芯片互联，10^18 FLOPS

TPU 的核心教训：

针对具体领域（ML 推理/训练）做专用架构，可以在 1% 的功耗范围内做到 100% 的合理性能。

7.4 经典 DSA 案例 2：NVIDIA GPU

虽然 GPU 也属于 DLP 架构，但 NVIDIA 的 GPU 在 ML 领域的成功使其成为 DSA 的重要案例。

GPU 作为 DSA

通用计算 (GP-GPU) ML DSA (Tensor Core)
通用 CUDA 核心 专用的 4×4 矩阵乘加单元
FP32 精度 FP16/BF16/INT8
通用存储层次 优化的张量存储
复杂调度 简化调度

Tensor Core (2017, Volta)：

  • 每个 Tensor Core 每个周期完成 4×4×4 矩阵乘法 + 加法
  • 一个 SM 中有 8 个 Tensor Core
  • 每个 SM 的峰值：8 × 4 × 4 × 4 = 512 FLOPS/周期 (FP16)

GPU vs TPU 的架构对比

特性 NVIDIA GPU (V100) Google TPU v3
计算核心 5120 CUDA + 640 Tensor Core 2× 128×128 Systolic Array
精度 支持 9 种精度 BF16
片上存储 6MB L2 + 16MB 共享内存 32MB
板级存储 32GB HBM2 32GB HBM2
互联 NVLink (8 chips) 自定义环网
灵活性 高（通用 CUDA） 低（ML 专用）
应用 训练+推理+通用计算 ML推理（v1），训练（v3）

7.5 FPGA 与 ASIC 的权衡

特性 FPGA ASIC
设计成本 低（无需光罩） 极高（7nm 光罩 > $5M）
单位成本 中（FPGA 本身贵） 极低（大量出货后）
性能 中（ASIC 的 10–20%） 最高
功耗 高（ASIC 的 3–5x） 最低
灵活性 可重配置 固定功能
上市时间 快 12–24 个月

什么时候用 FPGA：

  • 中等量（< 10K 台）
  • 标准/协议频繁变化
  • 软件开发与硬件开发并行

什么时候用 ASIC：

  • 大量出货（> 100K 台）
  • 对功耗/性能要求极致
  • 功能固定

微软 Catapult 项目：在数据中心服务器中插入 FPGA，用于：

  • Bing 搜索排名加速（2x 延迟改进）
  • 网络加速（低延迟 RDMA）
  • 可动态重配置适应不同负载

7.6 DSA 的设计方法论

设计流程 

应用分析 → 瓶颈识别 → 架构探索 → 模拟验证 → 实现
  1. 应用分析：理解目标负载的计算模式、数据流、访存特征
  2. 瓶颈识别：Amdahl 定律分析，找出最耗时的部分
  3. 架构探索：设计空间探索（并行度、精度、存储大小、带宽）
  4. 模拟：Cycle-approximate 模型验证性能
  5. 实现：RTL 设计、综合、物理设计

罗瑟塔石方法

Hennessy & Patterson 提出的 DSA 设计方法：

  1. Trial-and-Error ISAs：先做功能模拟器，迭代 ISA 设计
  2. HLL to ISA：高级语言到 ISA 的映射效率是关键
  3. Compiling Recording：编译器可以记录典型程序的执行模式
  4. ISA Simplification：去除不需要的复杂指令
  5. Continuous Co-design：SW/HW 协同设计，硬件和软件一起迭代

DSA 的 Roofline 模型

Roofline 模型：可视化分析性能瓶颈的工具。

性能
  ↑                     ┌─── 计算上限 (峰值 FLOPS)
  │                     │
  │      ┌──────────────┤
  │     /               │
  │    /                │
  │   /                 │
  │  /                  │
  │ / 内存带宽上限      │
  └──────────────────────────────→ 运算强度 (ops/byte)
  • 运算强度低 → 内存带宽瓶颈（左下方）
  • 运算强度高 → 计算瓶颈（右上方）
  • “Ridge Point”：从带宽受限到计算受限的转折点

7.7 DSA 的功耗分析

功耗/性能权衡

能量延迟积 (Energy-Delay Product, EDP)：

EDP=Energy×DelayEDP = Energy \times DelayEDP=Energy×Delay
  • 通用 CPU：EDP 最高（控制逻辑多、缓存大）
  • GPU：EDP 中等（SIMT 架构，控制简单）
  • DSA：EDP 最低（专用化程度最高）

专用化的能效收益来源

收益来源 原理 典型收益
更少控制逻辑 无需分支预测/乱序 3–5x
更窄的数据宽度 使用特定精度 2–10x (INT8 vs FP32)
更简单的存储 Scratchpad vs 缓存 2–3x
规则数据访问 预取、流式 2–5x
并行度匹配 设计满足负载需求 2–4x

总收益：在合适领域，10–100x 的每瓦性能提升。

7.8 DSA 的未来趋势

可重构 DSA

  • CGRA (Coarse-Grained Reconfigurable Array)：粗粒度可重构阵列，介于 FPGA 和 ASIC 之间
  • 可重构数据路径：运行时可配置的计算和数据流
  • 形态可变的加速器：在不同模式下切换

3D 集成与近存计算

  • HBM + 逻辑：3D 堆叠内存减少数据搬运
  • 近内存计算 (Near-Memory Computing)：在内存附近做计算
  • 存内计算 (Processing-in-Memory, PIM)：在内存阵列内部做计算

暗硅 (Dark Silicon) 与 DSA

随着工艺微缩，芯片中能同时工作的晶体管比例越来越小（暗硅）。DSA 通过：

  • 空间复用（不同专用单元在不同时间工作）
  • 掉电管理（不用的单元完全关闭）
  • 异构集成（多种专用核心 + 少量通用核心）

利用暗硅问题——与其让晶体管闲着，不如用来做专用加速器。

🧓
暗硅是你在做低功耗设计时实际面对的问题——为什么有那么多 power domain？为什么要加电源门控？因为工艺越先进，漏电越大，不用的模块必须彻底断电，否则静态功耗就会吃掉整个功耗预算。 DSA 的设计哲学和你的低功耗经验是一致的：只在需要的时候、用最有效的方式、做特定的事情。

7.9 常见谬误与陷阱

  1. 认为 DSA 适用于所有应用：DSA 在通用计算场景效果不佳，需要领域足够大才有 ROI
  2. 低估设计成本：ASIC 的 NRE 成本极高，需要足够产量分摊
  3. 忽略软件生态：没有好的编译器/SDK，DSA 再强也难推广
  4. 过度专用化：太专一可能导致需求变化时硬件作废
  5. 认为 DSA 永远是 ASIC：FPGA 和 CGRA 在灵活性和性能之间提供了有用折中
  6. 低估数据搬运的开销：加速器再快，数据送不进去也是白搭
  7. 忽视热密度问题：加速器的高利用率可能导致热密度集中在局部区域

7.10 总结

本章核心要点：

  1. DSA 是后 Dennard 时代的主要性能增长引擎——通过专用化大幅提升每瓦性能
  2. 三个设计原则：专用存储层次、窄数据路径、最少控制逻辑
  3. Google TPU 是 DSA 的经典案例——Systolic Array + 专用存储，在推理中实现 30–50x 的每瓦性能提升
  4. GPU 从通用计算向 ML DSA 演进（Tensor Core 的引入）
  5. Roofline 模型是 DSA 性能分析的基本工具，用于识别带宽还是计算瓶颈
  6. DSA 与传统 CPU 互补——异构 SoC (CPU + GPU + NPU) 是未来趋势
  7. 暗硅 + DSA：让晶体管做有用的事，而不是闲着
