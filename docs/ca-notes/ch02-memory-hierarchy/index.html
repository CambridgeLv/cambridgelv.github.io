计算机体系结构学习笔记 — 第2章 存储层次

原著: Computer Architecture: A Quantitative Approach, 6th Edition — John L. Hennessy & David A. Patterson

存储层次是计算机体系结构中最核心的设计问题之一。本章详细分析缓存设计、主存组织和虚拟内存技术。

2.1 引言 — 存储墙与性能瓶颈

为什么需要存储层次

处理器-内存性能差距是体系结构面临的最根本矛盾之一：

技术 1980 2015 年均改进
处理器性能 1 ~10,000 ~35%/年
DRAM 延迟 1 ~2 ~1%/年
DRAM 容量 1 ~1,000 ~20%/年

这个不断扩大的差距被称为存储墙 (Memory Wall)。处理器的计算能力每 18 个月翻倍，但主存延迟的改进每年不到 5%。这意味着一次主存访问的代价在相对性能中的权重越来越大。

🧓
你在做 STA 时应该体会过——综合报告里 critical path 经常是那些经过 SRAM/cache 的路径。缓存的 timing 收敛问题本质上是存储墙在物理实现层面的体现。架构上如果不是多用本地寄存器、少穿缓存和主存，你后面的时序根本收不拢。

存储层次的基本思想

利用局部性原理（第 1 章 1.9 节）：

  • 时间局部性：最近访问的数据很可能再次被访问
  • 空间局部性：访问一个地址后，相邻地址很可能被访问

存储层次通过在快但小和大但慢之间做 trade-off，用成本合理的方案提供接近最快的访问速度：

寄存器      (1 cycle,  ~1KB)
    ↓       L1 缓存    (2–4 cycles, 32–64KB)
    ↓       L2 缓存    (10–20 cycles, 256–512KB)
    ↓       L3 缓存    (30–50 cycles, 2–32MB)
    ↓       DRAM 主存  (100–300 cycles, 8–64GB)
    ↓       磁盘/SSD   (10⁶–10⁷ cycles, 1–10TB)

层次化存储的关键原则：

  1. 包含性 (Inclusion)：上层内容往往是下层的一个子集
  2. 越往上越快、越小、越贵（每字节）
  3. 越往下越慢、越大、越便宜
  4. 数据以块 (block/line) 为单位移动

2.2 缓存基础知识

缓存的结构

缓存的基本操作单元是缓存行 (cache line/block)。当 CPU 请求一个地址时：

  1. 缓存命中 (hit)：数据在缓存中 → 快速返回
  2. 缓存缺失 (miss)：数据不在缓存中 → 从下一级获取，加载到本层

缓存的三个关键设计参数

Q1: 块 (Block) 放在哪里？— 放置策略

策略 描述 优点 缺点
直接映射 每个块只能放在唯一位置 简单、快 冲突缺失率高
全相联 每个块可放在任意位置 冲突缺失最低 硬件复杂、慢
组相联 (n-way) 每个块可放在 n 个位置之一 折中方案 -

组相联是现代处理器最常用的策略。对于 N 路组相联缓存：

  • 内存地址划分成：标记 (tag) | 组索引 (index) | 块内偏移 (offset)
  • 每组 (set) 包含 N 个缓存行
  • 一个块可以映射到对应组的 N 个位置中的任意一个

组相联的硬件实现：比较器数量 = 相联度 N，用于并行比较所有候选行的 tag。

Q2: 如何找到块？— 查找机制

  1. 直接映射：索引字段直接定位 → 比较 tag
  2. 组相联：索引字段定位组 → 并行比较 N 个 tag
  3. 全相联：无索引 → 并行比较所有 tag（只在小缓存中使用）

Q3: 缓存缺失时替换哪一块？— 替换策略

策略 描述 实现难度 效果
LRU (最近最少使用) 替换最长时间未访问的 复杂 最佳（实践中）
FIFO 替换最早进入的 简单 较好
随机 随机选一个替换 最简单 出乎意料地好
伪 LRU (PLRU) 近似 LRU，用位追踪 适中 接近 LRU

LRU 的硬件实现：N 路组相联需要 $\log_2(N!)$ 位来追踪最近使用顺序。2 路只需要 1 位，4 路需要 ~5 位（常用 Pseudo-LRU）。

写策略

写命中

策略 描述 优点 缺点
写通 (Write-through) 数据同时写入缓存和下一级 一致性简单 带宽浪费
写回 (Write-back) 只写缓存，标记 dirty，逐出时写回 节省带宽 一致性复杂

写缓冲 (Write buffer)：写通策略中，为了隐藏写主存的延迟，使用写缓冲暂存写入数据。

🧓

综合时缓存的设计直接决定了时序和面积：

  • 写通：每次写都要穿透到下一级，时序路径更长
  • 写回：带宽效率高，但 dirty bit 逻辑增加了状态机复杂度 做 RTL 时这两种策略的差别很明显，写回的 FSM 通常比写通复杂一个数量级。

写缺失

策略 描述
写分配 (Write-allocate) 先加载块到缓存，再写（与写回搭配）
非写分配 (No-write-allocate) 直接写到下一级，不加载到缓存（与写通搭配）

其中 AMAT (Average Memory Access Time) 是平均内存访问时间。

缺失率 (Miss Rate) 与缺失惩罚 (Miss Penalty)

  • 缺失率 = 缓存缺失次数 / 总访问次数
  • 缺失惩罚 = 一次缓存缺失导致的额外延迟（从下一级读取数据的时间）

典型数据（以 64KB L1 缓存为例）：

缓存类型 命中时间 缺失率 缺失惩罚 (到 L2)
L1 指令缓存 1–2 cycles 1–5% 10–20 cycles
L1 数据缓存 2–4 cycles 5–10% 10–20 cycles
L2 统一缓存 10–12 cycles 1–5% 100–300 cycles (到 DRAM)

2.3 缓存性能的十种优化方法

原著将缓存优化归纳为六个类别、十种方法：

第一类：减少命中时间

1. 使用小且简单的 L1 缓存

使用直接映射或低相联度的小缓存来减少命中时间。

  • 优点：tag 比较逻辑简单，访问延迟低
  • 缺点：冲突缺失增多
  • 现代趋势：L1 通常 2–4 路组相联，32–64KB

2. 路预测 (Way Prediction)

根据上次访问的路来猜测这次可能在哪一路，先访问该路，猜错时再访问其他路。

  • 减少平均命中时间
  • 常用于嵌入式处理器的高相联缓存

3. 流水线化缓存访问

将缓存访问拆成多个流水级，允许更高频率。

  • 常见于乱序执行处理器
  • 缺点：流水线停顿增加分支误预测惩罚

第二类：增加缓存带宽

4. 多端缓存 (Multi-banked Cache)

将缓存分成多个独立 bank，可同时服务多个访问。

  • L1 数据缓存通常是多端口的（2–4个读/写端口）
  • 物理实现代价很大（面积 × 端口数）

5. 无阻塞缓存 (Non-blocking Cache)

缓存缺失时不阻塞后续的缓存命中访问。

  • 全空 (hit under miss)：缺失进行中，允许命中继续
  • 全空多缺失 (hit under multiple miss)：同时处理多个未完成的缺失，需要 MSHR (Miss Status Holding Registers)
🧓
无阻塞缓存对乱序处理器非常关键。你在综合时如果看到某个 SRAM 上有大量 arbitration logic，很可能就是在做 multi-bank 或 non-blocking 的支持。RTL 里 MSHR 的实现是个典型的状态机设计问题。

第三类：减少缺失率

6. 增大块大小

  • 增大块大小利用空间局部性 → 减少缺失率
  • 但过大会增加缺失惩罚（传输时间增加）和污染（不需要的数据挤走了有用的）

7. 增加缓存容量

  • 简单逻辑：更大的缓存 → 更低的缺失率
  • 但命中时间增加、面积和功耗增加
  • 命中时间和容量的 trade-off 是 L1/L2/L3 划分的根本原因

8. 高相联度

  • 增加相联度 → 减少冲突缺失
  • 但命中时间增加（需要更多的 tag 比较）

第四类：通过预取减少缺失惩罚或缺失率

9. 硬件预取 (Hardware Prefetching)

根据访问模式预测下一个要访问的地址，提前加载到缓存。

  • 顺序预取：预测下一个连续地址（适合数组遍历）
  • 步长预取：识别固定步长的访问模式
  • 指针预取：追踪指针链预测

指令预取：顺序执行时直接取下一个缓存块，简单有效。

数据预取：需要识别访存模式，更复杂。

🧓
你在做 RV32I 核心时，如果以后加上预取器，会发现预取逻辑对时序非常有挑战——因为它需要预测下一个地址的同时不影响主流水线的时序。预取深度和准确性之间的 trade-off 是个经典架构问题，也是很多专利的产出来源。

第五类：通过编译器减少缺失率

10. 编译器优化

优化 原理 效果
循环交换 (Loop Interchange) 按内存布局方向遍历 利用空间局部性
循环融合 (Loop Fusion) 合并两个独立循环 提高数据复用
分块 (Blocking/Tiling) 分块处理，提高子块的缓存命中 显著减少缺失
数组对齐 确保数组起始在缓存行边界 避免行跨越
预取指令 插入 prefetch 指令 在数据使用前提前加载

// 简单的 ijk 矩阵乘法（缓存不友好）
for (i = 0; i < N; i++)
    for (j = 0; j < N; j++)
        for (k = 0; k < N; k++)

// 分块优化（缓存友好）
for (i = 0; i < N; i += B)
    for (j = 0; j < N; j += B)
        for (k = 0; k < N; k += B)
            for (ii = i; ii < min(i+B, N); ii++)
                for (jj = j; jj < min(j+B, N); jj++)
                    for (kk = k; kk < min(k+B, N); kk++)
                        C[ii][jj] += A[ii][kk] * B[kk][jj];

分块后的代码对缓存更友好——当 B 的大小选择合适时，子矩阵完全在缓存中，避免了反复从主存加载。

第六类：通过并行减少缺失惩罚

多级缓存：

  • L1 → L2 → L3 → DRAM
  • 各级缓存逐渐增大、逐渐变慢
  • 大多数缺失由 L2/L3 捕获，减少对主存的访问

2.4 缓存设计的六个基本问题总结

问题 选项
Q1: 块放到哪里？ 直接映射 / 组相联 / 全相联
Q2: 如何找到？ tag 比较 + 索引
Q3: 替换哪个？ LRU / 伪 LRU / 随机
Q4: 写策略？ 写通 / 写回
Q5: 块多大？ 16B–128B（典型 64B）
Q6: 缺失时做什么？ 阻塞 / 非阻塞（hit under miss）

2.5 虚拟内存

为什么需要虚拟内存

  1. 隔离：每个进程拥有独立的地址空间，互不干扰
  2. 简化编程：程序使用连续虚拟地址，无需关心物理内存碎片
  3. 共享：不同进程可通过虚拟地址共享物理页面
  4. 保护：操作系统控制页表的访问权限

分页 (Paging)

虚拟内存的块称为页 (page)，典型大小 4KB（也支持 2MB 大页）。

页表 (Page Table)：虚拟地址到物理地址的映射表。

  • 每个进程有自己的页表
  • 页表存储在物理内存中
  • 每次访存都需要查页表（！）

TLB — 页表缓存

如果每次访存都要查一次内存中的页表，等于每次访存变成两次访存。解决方案：TLB (Translation Lookaside Buffer) — 页表的硬件缓存。

Effective_Access_Time=Hit_Time+Miss_Rate×Miss_PenaltyEffective\_Access\_Time = Hit\_Time + Miss\_Rate \times Miss\_PenaltyEffective_Access_Time=Hit_Time+Miss_Rate×Miss_Penalty

TLB 缺失处理：

  • 硬件处理（x86, ARM）：MMU 自动遍历页表
  • 软件处理（MIPS）：TLB 缺失触发异常，OS 处理

典型 TLB 参数：

  • L1 TLB：64–128 项，全相联或组相联
  • L2 TLB：512–1024 项，组相联，更大的页支持
🧓
TLB 缺失的惩罚很大，因为需要去内存中遍历多级页表（4 级页表 = 4 次内存访问）。这就是为什么大页（2MB/1GB）在数据中心和 HPC 中这么重要——TLB 的一个条目能映射更大的地址空间，有效减少 TLB 缺失。 你做 STA 时可能遇到过 LVS 或综合工具因为内存映射问题变慢的情况，本质就是 TLB 缺失导致的性能损失。

保护与共享

  • 页表项包含权限位（读、写、执行、用户/内核）
  • 页表项包含用户态/内核态标志
  • 通过共享页面实现进程间通信

2.6 现代处理器的缓存层次

Intel Skylake 为例

缓存 大小 相联度 行大小 延迟 策略
L1 指令缓存 32KB 8路 64B 4 cycles -
L1 数据缓存 32KB 8路 64B 4 cycles 写回
L2 256KB 4路 64B 11 cycles 写回
L3 2MB/核 16路 64B 30–40 cycles 写回

L3 缓存是 LLC (Last-Level Cache)，所有核共享。

缓存一致性 (Cache Coherence) 概述

多核系统中，每个核有自己的私有 L1/L2 缓存，多个缓存副本可能指向同一物理地址。需要一致性协议保证数据一致性。

MESI 协议是最经典的缓存一致性协议：

状态 含义
M Modified 该行仅在本缓存，已被修改（脏），与主存不一致
E Exclusive 该行仅在本缓存，未被修改，与主存一致
S Shared 该行在多个缓存中存在副本，与主存一致
I Invalid 该行无效

监听协议 (Snooping)：每个缓存控制器监听总线上的访存请求，维护一致性。

目录协议 (Directory-based)：使用集中式目录跟踪每个缓存行的状态，适合大规模多核。

（缓存一致性将在第 5 章「线程级并行」中详细讨论。）

2.7 主存组织与 DRAM 技术

DRAM 基本操作

  • Row Access Strobe (RAS)：激活行
  • Column Access Strobe (CAS)：读取/写入列
  • 行缓冲区 (Row Buffer)：当前激活的行，可快速访问
  • 行刷新：DRAM 需要定期刷新（~64ms）以保持电荷

DRAM 访问延迟：

tRAS→tRCD→tCAS→tRPt_{RAS} \rightarrow t_{RCD} \rightarrow t_{CAS} \rightarrow t_{RP}tRAS​→tRCD​→tCAS​→tRP​
  • $t_{RAS}$ — 行激活 (Row Activate)
  • $t_{RCD}$ — 行到列延迟 (Row to Column Delay)
  • $t_{CAS}$ — 列访问 (Column Access Strobe)
  • $t_{RP}$ — 行预充电 (Row Precharge)

内存控制器

现代处理器的内存控制器集成在 CPU 内部（非传统北桥），每个内存控制器管理一个或多个 DIMM 插槽。

内存交织 (Memory Interleaving)：将连续地址分布到多个内存通道，并行访问提高总带宽。

2.8 虚拟内存与缓存的交互

TLB + 缓存的访问流程

  1. CPU 发出虚拟地址
  2. 同时查询 TLB（地址翻译）和 L1 缓存（虚拟/物理索引）
  3. TLB 返回物理地址，L1 缓存返回数据（命中）或转发请求到 L2
  4. L2 使用物理地址索引（物理标记）

VIPT vs PIPT vs VIVT：

索引方式 描述 别名问题
VIVT 虚拟索引 + 虚拟标记 有严重别名问题
VIPT 虚拟索引 + 物理标记 需页偏移对齐
PIPT 物理索引 + 物理标记 无别名，但慢

现代处理器普遍使用 VIPT L1 缓存（利用虚拟地址的页内偏移位做索引，在 TLB 查询的同时开始索引），L2/L3 使用 PIPT。

保护机制

  • 物理地址扩展 (PAE)：x86 的 36 位物理地址扩展
  • SMAP/SMEP：禁止内核执行用户代码、访问用户数据
  • ASLR：地址空间布局随机化，安全特性

2.10 常见谬误与陷阱

  1. 认为更大的缓存总是更好：更大的缓存延迟更高，可能在低命中率提升下反而降低性能
  2. 忽略索引冲突：某些特定的地址访问模式可能导致严重的冲突缺失（高速缓存颠簸）
  3. 假设缓存缺失惩罚是常数：非阻塞缓存和多级缓存使缺失惩罚的估算更复杂
  4. 忽视 TLB 缺失：大数据集的程序可能 TLB 缺失频繁，即使数据缓存命中率很高
  5. 误认为写通策略总是慢于写回：在有写缓冲的情况下，写通策略可能足够快且一致性更简单
  6. 忽略缓存和虚拟内存的交互：页面着色 (page coloring) 等技术在特定场景很重要
  7. 假设编译器优化对所有程序都有效：优化效果高度依赖程序的访存模式

2.11 总结

本章核心要点：

  1. 存储层次是所有计算机系统性能的关键——处理器等待内存是 CPU 时间中最主要的开销
  2. 局部性原理是存储层次设计的基础，所有缓存技术都在利用时间和空间局部性
  3. 缓存三问：放在哪？怎么找到？替换哪个？——所有缓存设计都围绕这三个问题
  4. 十种优化方法从六个维度提升缓存性能：减少命中时间、增加带宽、降低缺失率、预取、编译器优化、并行
  5. 虚拟内存提供安全隔离和地址空间抽象，TLB 是解决地址翻译开销的关键
  6. 多级缓存层次（L1→L2→L3→DRAM）在延迟和容量间做最优折中
  7. 缓存一致性（MESI）是多核系统的核心挑战，与虚拟内存交互共同形成复杂的内存系统
