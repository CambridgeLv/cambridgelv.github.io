本文覆盖 5 级 RISC 流水线的全部基础概念，与我们的 RV32I 核心项目直接对应。

C.1 流水线入门

什么是流水线

流水线（Pipelining）允许多条指令的执行在时间上重叠。每个步骤称为一个流水级（pipe stage）。

5 级 RISC 流水线：

流水级 功能
IF 取指令，PC+4
ID 译码，读寄存器堆，立即数生成
EX ALU 运算，分支条件判定
MEM 数据存储器访问
WB 结果写回寄存器堆

性能公式： $$Speedup = n \quad\text{(理想条件下，n 级流水线)}$$

实际限制：流水级不平衡、流水线寄存器开销、冒险。

三大冒险

类型 原因 解决方案
结构冒险 资源冲突 分离 I$/D$
数据冒险 RAW 依赖 转发+stall
控制冒险 分支/跳转 分支预测+flush

转发（Forwarding）

转发路径：EX/MEM → EX 输入，MEM/WB → EX 输入 无法转发的情况：load-use（需 stall 1 周期）

分支预测

策略 实现 误预测惩罚
Always Not Taken 默认 PC+4 2 cycles
2-bit 饱和计数器 4 状态 FSM 2 cycles

流水线寄存器

每个流水级之间插入：IF/ID、ID/EX、EX/MEM、MEM/WB

C.2 综合案例：MIPS R4000

8 级整数流水线（100–250 MHz），比 5 级深，分支惩罚增加到 3 cycles。 从 5 级到 8 级是频率 vs 复杂度的经典 trade-off。

C.3 记分牌算法

动态调度的前身：顺序发射、乱序执行、记分牌跟踪资源。 四个阶段：Issue → Read Operands → Execute → Write Result 局限：无寄存器重命名，被 Tomasulo 算法取代（见第 3 章）。

老大注

我们做的 RV32I 核心就是 5 级 RISC 流水线。综合时注意流水线寄存器之间的组合逻辑路径（如 ALU → Forward MUX → ALU）的时序约束。

加深流水线 → 频率更高 → 但分支惩罚更大。这就是架构师做 trade-off 的地方。
